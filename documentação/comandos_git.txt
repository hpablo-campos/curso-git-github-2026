### comandos básicos ###

git init.				 -> cria um repo git na pasta atual
git status				 -> mostra o status do trabalho
git add					 -> adiciona novos arquivos ou o estado de um arquivo já existente, ao staging area
git commit -m				 -> faz commit do que estava no staging area
git log					 -> exibe o historico de commits
git diff <nome-arquivo> 		 -> indica as alterações nos arquivos.
git restore --staged <nome-arquivo>	 -> remove um arquivo não salvo da staging area


### branches  ###

git checkout -b <nome-branch>	 -> cria uma nova branch e muda a branch atual de trabalho
git checkou <nome-branch>	 -> muda a branch atual de trabalho
git branch			 -> exibe as branches existentes no diretório de trabalho atual
git merge <branch alternativa>	 -> mescla todos os commits da branch alternativa com os da principal
git branch -d <nome-branch> 	 -> deleta uma branch (deve estar em uma branch diferente para deletar) 

#### pulls e pushes  ####


git remote add origin <origin-url> -> vincula um repositório local a uma origin remota (github, gitlab, etc.)
git push <remote> <branch> -> A maneira mais simples de fazer um 'push'. (git push origin main) vai empurrar a branch main local para a branch main remota.
git pull <remote> <branch> -> git pull origin main: puxa para o repo local, todas as atualizações da origin na branch main remota, para a branch main local.
